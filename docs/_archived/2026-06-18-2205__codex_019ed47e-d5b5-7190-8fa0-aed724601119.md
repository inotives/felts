## Key Decisions To Preserve

- Each new phase starts with `grill-with-docs`.
- Docs updates are committed and pushed to `main` before implementation starts.
- Implementation starts from a new feature branch based on latest `main`.
- EL code remains feature-based by source under `src/felts/sources/<source>/`.
- Shared flows are only for cross-source orchestration such as dbt transform and manual master flow.
- Raw completion event convention remains `felts.raw.<source>.<entity>.completed`.
- CSV source uses `source = csv_import`.
- CSV contracts live in `src/felts/sources/csv_import/contracts.yaml`.
- YAML contract keys are stable user-facing contract IDs.
- CSV extractor should be generic and use Python's standard library `csv` module.
- CSV dataset-specific params belong in YAML: source, entity, path pattern, encoding, delimiter, headers, identity strategy, raw identity fields, observed column, and dbt selector.
- Supported Phase 05 identity strategies: `filename_pattern` and `value_column_header`.
- CSV import requires `input_uri`; no directory scanning, globbing, or multi-file import in Phase 05.
- Local absolute paths, repo-relative paths, and `file://` URIs are allowed.
- `s3://`, `gs://`, `http://`, and `https://` are out of scope.
- Header mismatch fails before raw load.
- Row-level value issues can be preserved as invalid raw rows.
- Raw payload preserves CSV row strings and includes `_felts` trace metadata.
- `row_number` means physical file line number.
- Raw completion events for CSV should fire only when at least one valid raw row is inserted.
- Re-importing the same file should insert zero duplicate rows when row identities already exist.
