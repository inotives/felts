## Key Decisions To Preserve

- EL code should be feature-based by source: `src/felts/sources/<source>/`.
- Shared flows are only for cross-source orchestration such as dbt transform and manual master flow.
- Raw source schemas use provider schemas, for example `coingecko`.
- Staging, intermediate, and mart models for the same provider should live in the same provider schema.
- Raw completion event convention: `felts.raw.<source>.<entity>.completed`.
- Emit raw completion events only when the raw load succeeds and inserts rows.
- `coins_markets` remains registered with no schedule for now.
- Non-secret Felts defaults belong in `config.yaml`; secrets and machine-specific overrides belong in `.env`.
- Current settings precedence: explicit args > env vars > `.env` > `config.yaml` > code defaults.
